# Пояснительная записка: структура и наполнение

## Тема
Разработка распределенной системы мониторинга и корреляции событий производительности и безопасности Linux-серверов на базе Node.js.

## Что уже реализовано в проекте
- Распределенная архитектура `agent -> server -> dashboard`
- Сбор метрик CPU, памяти, диска, сети и процессов
- Прием бинарных TCP-сообщений сервером и сохранение данных в PostgreSQL
- Корреляция событий и генерация alert-уведомлений
- Веб-интерфейс с обзором, карточками агентов, событиями, alert-уведомлениями и security-страницами
- Docker Compose стенд с несколькими демонстрационными Linux-агентами
- Автотесты и smoke-проверка контейнерного запуска

### 1. Обзор предметной области и требования к программному обеспечению
#### 1.1 Предметная область
- Мониторинг производительности серверов Linux
- Контроль событий безопасности
- Централизованный сбор и визуализация данных

#### 1.2 Основные функциональные требования
- Сбор системных метрик на удаленных узлах
- Передача данных на центральный сервер
- Хранение истории метрик и событий
- Корреляция аномальных и подозрительных событий
- Вывод результатов через единый web-dashboard

#### 1.3 Нефункциональные требования
- Масштабируемость по числу агентов
- Отказоустойчивое восстановление соединения
- Расширяемость протокола и типов отчетов
- Контейнеризация и воспроизводимое развертывание

### 3. Проектирование и реализация программного обеспечения
#### 3.1 Общая архитектура системы
- Трехуровневая схема: agent, server, dashboard
- Потоки данных между слоями
- Выбор клиент-серверной модели

#### 3.2 Реализация agent-части
- Сбор метрик из `/proc` и системных утилит
- Многопоточность через `worker_threads`
- Разделяемая память `SharedArrayBuffer`
- Реализация `Mutex` и `Semaphore`
- Обработка системных сигналов

#### 3.3 Реализация server-части
- TCP receiver
- Декодирование бинарного протокола
- Сохранение данных в PostgreSQL
- REST API и WebSocket-уведомления
- Корреляция событий и генерация alert-уведомлений

#### 3.4 Реализация dashboard-части
- Обзор состояния инфраструктуры
- Просмотр агентов и метрик
- Просмотр security-событий
- Детализированные security-отчеты по узлам

#### 3.5 Реализация дополнительных security-модулей
- SSHD audit
- Open port scan
- Firewall audit
- Hardening score
- Login activity analysis

#### 3.6 Использование средств системного программирования
- Сокеты TCP
- Worker threads
- SharedArrayBuffer и Atomics
- Mutex/Semaphore
- Child processes
- Signals

#### 3.7 Система контроля версий и организация разработки
- Использование Git и удаленного репозитория
- Ветки `main`, `develop`, `feature/*`
- Изоляция задач и пошаговые коммиты

#### 3.8 Контейнеризация
- Compose-конфигурация
- Связка `postgres + server + dashboard + agents`
- Особенности демонстрационного стенда
